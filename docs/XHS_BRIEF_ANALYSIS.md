 ## 一、原始概念复述
 
 ### 1.1 背景
 
 原帖作者的女朋友想换新 iPhone，但总是攒不下钱。不是乱花大钱，而是奶茶、外卖、小东西——月底一看，离想买的东西还是很远。
 
 ### 1.2 核心洞察
 
 > "花钱有快感，存钱只有一个冷冰冰的数字。"
 
 存钱这件事 **没有反馈**。传统进度条和数字无法提供消费行为带来的即时满足感。用户需要的是 **"看得见的进度"**，而不是一个抽象的数字。
 
 ### 1.3 创意方案
 
 作者做了一个原型：
 
 - 用户选择一样想买的东西（例：新 iPhone）
 - **每存一笔钱，屏幕里的 iPhone 就会拼出来一点**
 - 碎片是 **3D 的，会从旁边飞回来，慢慢合成完整的样子**
 - 第一次看到时反馈："这个比进度条有感觉多了"
 
 ### 1.4 用户测试结果
 
 原帖发出后，正在找内测用户。帖子评论区反响热烈，说明很多人"不是缺少存钱道理，只是缺少一个能让自己'看见进度'的东西"。

 ## 四、主视觉效果详解
 
 ### 4.1 V1 版 — WishObjectView（保留可回退）
 
 #### 视觉结构
 
 ```
 ┌─────────────────────────────┐
 │        黑色手机边框            │
 │  ┌───────────────────────┐  │
 │  │  ██▓▓░░░░░░░░░░░░░░░░ │  │  ██ = 已解锁（粉橙色渐变格子）
 │  │  ██▓▓░░░░░░░░░░░░░░░░ │  │  ░░ = 未解锁（低透明度）
 │  │  ████████░░░░░░░░░░░░ │  │
 │  │  ██████████░░░░░░░░░░ │  │  总计 8 行 × 5 列 = 40 格
 │  └───────────────────────┘  │
 └─────────────────────────────┘
 ```
 
 #### 动画效果
 
 | 触发时机 | 效果 | 实现方式 |
 |---------|------|---------|
 | 存钱触发新碎片解锁 | 格子 `scale(0) rotate(-180deg)` → `scale(1.15) rotate(8deg)` → `scale(1) rotate(0deg)` 弹出 | CSS `puzzle-pop-in` keyframe, 0.55s ease-out |
 | 格子依次飞入 | `animationDelay = col * 0.04 + row * 0.03` s，从左到右递进 | JS 内联 style |
 | 金额增温（无新碎片） | 光点从底部飞升 + 全机粉色径向发光 | Framer Motion + CSS 渐变 |
 | 金额变化 | 数字 `scale[1,1.06,1]` + 颜色黑→粉→黑 | Framer Motion animate |
 
 #### 风格特征
 
 - 手机边框：深色渐变（`#1c1917 → #292524`），科技感
 - 已解锁格子：粉橙渐变高亮（`from-pink-400 to-orange-400`）
 - 未解锁格子：`rgba(255,255,255,0.08)` 极低可见度
 - 适合：喜欢明显颗粒感和"填满"视觉的用户

 | 触发时机 | 效果 | 实现方式 |

 | 小金额存入（未解锁碎片） | 金色光点从底部飞入：`translateY(20px → -100px)`，`scale(0.4 → 0.8)` | CSS `tiny-coin-fly` keyframe, 1s |
 | 下一块碎片增温 | 手机外阴影变粉 + 底部闪烁"💫 存进去了" | Tailwind 条件类 + animate-pulse |

 - 手机边框：暖白/玫瑰金渐变（`#fef3f2 → #fff7ed`），柔和感
 - 屏幕底图：粉橙渐变壁纸始终存在
 - 未解锁区域：`bg-white/60` 半透明白色蒙层
 - 已解锁区域：雾层完全消失，露出完整壁纸
 - 整体观感：像一张照片从雾中逐渐显影
 - 适合：追求小红书感、温暖氛围的风格

 pieceValue = 10000 / 40 = 250  // 每块碎片价值

 ## 七、与原始概念的差异与取舍
 
 | 原帖 "3D 飞入" | 实际实现 | 取舍原因 |
 |----------------|---------|---------|
 | 3D 碎片从旁边飞入合成 | 2D 雾层 fade-out / 格子 pop-in | 避免 Three.js，首版轻量快速验证 |
 | 碎片有可见运动轨迹 | 碎片在原始位置 fade-in | 雾层 fade+blur+scale 产生"显影"感受 |
 | 任意目标物品 | 固定为手机形状 | MVP 锁定一种形态，后续自定义 |
 | 3D 空间感 | blur + scale 模拟"从远到近" | 轻量方式达到类似 3D 效果 |
 | 真实支付 | 手动记录 | 降低合规成本，聚焦体验 |

 ## 十一、V3 升级：飞行碎片奖励动效
 
 ### 11.1 升级动机
 
 V2 从硬网格升级为柔和雾层显影，视觉大幅改善。但对照小红书原帖体验，仍然缺少一个关键爽感：**"碎片从旁边飞回来，慢慢合成完整的样子"**。
 
 V2 更像"雾层消失，手机显影"；小红书原型更像是"我存钱后，碎片真的被召唤回来"。
 
 ### 11.2 新增组件：FlyingShards
 
 新增 `components/FlyingShards.tsx`，在 V2 手机视觉上层叠加一个 **2.5D 飞行碎片动效层**。
 
 | 属性 | 说明 |
 |------|------|
 | `triggerKey` | 每次存钱递增，驱动组件重新挂载播放动画 |
 | `mode` | `"warmup"` / `"unlock"` / `"complete"` 三种模式 |
 | `count` | 可选，覆盖默认碎片数量 |
 
 ### 11.3 三种动效模式
 
 #### warmup（小金额存钱，未解锁碎片）
 
 - **触发条件**：当前存入金额不足以解锁一块新碎片（< ¥250）
 - **碎片数量**：2 个
 - **视觉**：金色/琥珀色小圆点，带金色辉光，从手机下方飞入屏幕中心
 - **动画**：`opacity[0→1→0]`、`translateY` 从底部升入、`scale[0.3→1.3→0.8]`
 - **风格**：像两枚小金币被投进手机
 
 #### unlock（解锁新碎片）
 
 - **触发条件**：本次存钱解锁了 1 块或更多碎片
 - **碎片数量**：4 + min(新增碎片数, 4) = 最多 8 个
 - **视觉**：粉橙渐变小方块，边缘带粉色调辉光，从手机四周（两侧/下方）飞向屏幕中心
 - **动画**：`opacity[0→1→0]`、`translate` 从外侧飞入、`scale[0.3→1.3→0.8]`、`rotate(-20°→0°)`
 - **blur**：0.5px 轻微模糊增加发光感
 - **风格**：像愿望碎片被召唤回来，拼入手机
 
 #### complete（100% 完成）
 
 - **触发条件**：进度到达 100%
 - **碎片数量**：12 个
 - **视觉**：✨ 🌟 ⭐ 💫 emoji 从手机屏幕中心向四周散开
 - **动画**：`opacity[0→1→0]`、`translate` 向外辐射、`scale[0→0.6→1.4]`、`rotate` 增加随机感
 - **风格**：像手机完成时"绽放"出星星
 
 ### 11.4 技术实现
 
 ```tsx
 // 每个碎片是一个 Framer Motion motion.div
 // 绝对定位在电话容器内（left: 50%, top: 50%）
 // 使用 key={triggerKey} 确保每次存钱时组件重新挂载
 
 // 稳定配置数组（不产生随机数，避免 SSR/hydration 问题）
 const WARMUP_SHARDS = [
   { x: -20, y: 80, rotate: -10, delay: 0,    size: 8 },
   { x:  22, y: 90, rotate:   8, delay: 0.12, size: 7 },
 ];
 
 // unlock/complete 使用更丰富的数组（最多 8-12 个碎片）
 ```
 
 每个碎片动画使用 Framer Motion keyframes + spring cubic-bezier 贝塞尔曲线（`0.34, 1.56, 0.64, 1`），产生 **回弹/过冲** 效果，增强"碎片飞入"的物理感。
 
 ### 11.5 完成态增强
 
 在 V2 已有基础上增加：
 
 - **完成徽章**：手机右上角"已完成"胶囊标签（粉橙渐变 + 白色文字）
 - **额外星星**：增加第 4 颗浮动星 `✨`（左下角，3s 周期，0.8s 延迟）
 - **完成横幅**：庆祝文案下方增加"共积攒 ¥10,000" 微光胶囊条
 - **complete 飞行碎片**：12 个星星从手机中心向外散开
