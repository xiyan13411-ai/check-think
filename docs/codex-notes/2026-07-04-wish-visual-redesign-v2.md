# Codex 同步说明：愿望物品主视觉 V2 重构

> 建议路径：`docs/codex-notes/2026-07-04-wish-visual-redesign-v2.md`  
> 适用仓库：`xiyan13411-ai/check-think`  
> 当前状态：已从抽象拼图格子升级为手机模型，但真机视觉反馈为“太丑、生硬”  
> 本次目标：从“CSS 手机网格”升级为更柔和、更像小红书截图的愿望物品揭示视图。

---

## 1. 背景

上一轮已经新增 `WishObjectView.tsx`，把原本的 8×5 抽象格子替换成了手机模型。

但是用户真机反馈是：

> 太丑了，也很生硬。

从截图看，当前版本的问题不在业务逻辑，而在视觉表达。

当前视觉更像：

- 黑色手机外壳
- 粉橙方格填满屏幕
- 大块网格拼图
- 演示 Demo 感
- 像小游戏原型，而不是小红书质感产品

这与目标“小红书那种存钱爽感”仍有差距。

---

## 2. 当前截图问题诊断

### 2.1 手机太黑太硬

当前手机机身使用了强黑色边框，视觉很重。  
在奶油色、小红书风背景里，黑色大块会显得突兀、廉价、像调试组件。

应改为：

- 奶油白 / 暖灰 / 玫瑰金边框
- 柔和阴影
- 玻璃高光
- 减少纯黑面积

---

### 2.2 屏幕网格太像“填色板”

当前 5×8 碎片是规则方块，线条明显，像表格或小游戏棋盘。

问题：

- 不像手机
- 不像愿望物品
- 不像“拼回来”
- 更像“把格子涂满”

应改为：

- 减弱网格线
- 用圆角碎片或柔和 mask
- 已解锁部分像“物品显影”
- 未解锁部分像“雾化/灰色轮廓”
- 不要让 40 块格子成为主视觉本身

---

### 2.3 100% 没有完成奖励

截图中已经是 100%，但手机仍然是格子状态，没有“完整物品出现”的惊喜。

100% 时应该有明显 payoff：

- 完整手机显现
- 屏幕变成完整渐变/壁纸
- 出现完成徽章
- 彩带/星星/光晕
- 文案更有仪式感

---

### 2.4 页面层级像工具，不像内容

目前视觉层级是：

1. 金额卡片
2. 黑色手机网格
3. 存钱按钮

但小红书感应该更像：

1. 一个漂亮愿望物品
2. 一句可以截图的状态文案
3. 存钱动作像“投喂”
4. 金额是辅助信息，不是财务报表

---

### 2.5 动效可能是“有动”，但不“爽”

当前 pop-in、warm glow 都属于技术动画。  
但爽感不是“元素动了”，而是：

- 动作和结果有因果
- 用户能看到钱被吸入目标
- 目标物品明显变完整
- 完成时有奖励高潮

---

## 3. 本轮设计原则

本轮不要继续优化“手机网格”。

请转向：

> 物品显影 / Reveal，而不是格子填色。

核心视觉从：

```txt
手机里有 40 个方格
```

改成：

```txt
一台漂亮的愿望手机正在从雾里显现
```

---

## 4. V2 视觉方向

### 4.1 新组件建议

新增或重构为：

```txt
components/WishPhoneReveal.tsx
```

可以保留 `WishObjectView.tsx` 作为 fallback。

推荐在 `app/page.tsx` 中用 `WishPhoneReveal` 替代 `WishObjectView`。

---

### 4.2 主视觉：柔和手机显影

视觉目标：

- 不使用纯黑大外壳
- 手机更像产品海报里的物品
- 屏幕主体是一整块柔和渐变
- 未解锁区域覆盖半透明雾层
- 已解锁区域逐步揭开
- 碎片可以存在，但不要像硬格子

结构建议：

```txt
外层愿望舞台 card
  - 柔和粉橙光晕
  - 浮动小星星/金币
  - 中央手机物品

手机模型
  - 暖白/玫瑰金边框
  - 圆角玻璃屏
  - 顶部灵动岛/摄像头弱化
  - 屏幕完整渐变壁纸
  - 覆盖 reveal mask
```

---

### 4.3 Reveal 方式

不要让用户看到强烈的 40 格线条。

仍然可以保留 40 个逻辑碎片，但视觉上做成：

- 更淡的边界
- 圆角小块
- 透明遮罩
- 不明显的 shard
- 已解锁区域让底部完整渐变显现

实现方式：

```tsx
<div className="phone-wallpaper">
  {/* 完整渐变背景 */}
</div>

<div className="reveal-grid">
  {pieces.map(piece => (
    <div className={unlocked ? "opacity-0" : "opacity-70 bg-cream-blur"} />
  ))}
</div>
```

也就是说：

- 底下始终是一张完整“愿望手机屏幕”
- 未解锁碎片是盖在上面的雾层
- 存钱后雾层减少
- 不是把粉色块一块块涂上去

这会比当前“方格填充”高级很多。

---

### 4.4 小金额反馈

小金额不解锁碎片时：

- 不要只闪方格
- 做“钱被吸进手机”的感觉

建议：

1. 一个小金币 / 光点从存钱按钮方向飞向手机
2. 手机整体轻轻发光
3. 下一块雾层边缘有一点亮
4. 下一块进度条保留，但要更轻

文案可保留：

```txt
这笔钱正在把下一块碎片推近
```

但视觉上要让用户看到“投喂感”。

---

### 4.5 大额反馈

大额解锁多个碎片时：

- 多个雾层依次消失
- 底下完整屏幕显现更多
- 手机周围爆一点星星/小彩带
- 不要每块都变成大方格

---

### 4.6 100% 完成状态

100% 时不要再显示拼图网格。

100% 应切换到完成态：

```txt
完整手机
亮屏
柔和光晕
🎉 愿望全部拼回来了！
```

可以加：

- 整机轻微浮动
- 背后径向光
- 星星粒子
- 完成徽章

验收标准：

> 100% 截图应该像一张完成海报，而不是一块填满的表格。

## 5. Codex 需要执行的任务

### Task 1：新增 WishPhoneReveal 组件

新增：

```txt
components/WishPhoneReveal.tsx
```

Props：

```ts
type WishPhoneRevealProps = {
  totalPieces: number;
  unlockedPieces: number;
  currentAmount: number;
  targetAmount: number;
  newlyUnlockedPieceIndexes?: number[];
  warmUpNextPiece?: boolean;
};
```

要求：

- 使用柔和手机模型
- 使用完整渐变屏幕作为底图
- 使用 5×8 雾层 overlay 作为“未解锁遮罩”
- 已解锁碎片对应 overlay 消失
- 新解锁碎片用 fade / scale / blur 动画消失
- 小金额时有光点飞入
- 100% 时进入完整完成态
- 保留下一块碎片进度，但样式更轻

---

### Task 2：替换当前 WishObjectView

在 `app/page.tsx` 中：

- 引入 `WishPhoneReveal`
- 替换当前 `WishObjectView`
- 保留 `WishObjectView` 文件作为 fallback
- 不改存钱逻辑
- 不改 localStorage
- 不改成就逻辑
- 不改历史记录逻辑

---

### Task 3：调整视觉样式

新增 CSS 动画：

```css
@keyframes reveal-fog-away {
  0% {
    opacity: 0.75;
    filter: blur(0px);
    transform: scale(1);
  }
  60% {
    opacity: 0.25;
    filter: blur(6px);
    transform: scale(1.08);
  }
  100% {
    opacity: 0;
    filter: blur(10px);
    transform: scale(0.92);
  }
}

@keyframes phone-soft-glow {
  0%, 100% {
    filter: drop-shadow(0 18px 35px rgba(244, 114, 182, 0.16));
  }
  50% {
    filter: drop-shadow(0 24px 55px rgba(244, 114, 182, 0.38));
  }
}

@keyframes tiny-coin-fly {
  0% {
    opacity: 0;
    transform: translateY(36px) scale(0.5);
  }
  35% {
    opacity: 1;
  }
  100% {
    opacity: 0;
    transform: translateY(-120px) scale(0.9);
  }
}
```

不要让动画太多，安卓机要流畅。

完成后真机测试应满足：

- [ ] 页面第一眼不再显得丑/硬

- [ ] 不再像硬网格填色板
- [ ] 用户一眼能看出是愿望手机
- [ ] 存小钱时有“喂到手机里”的感觉
- [ ] 存大钱时能看到雾层消失 / 物品显现
- [ ] 100% 时是完整手机完成态
- [ ] 整体更像小红书截图
- [ ] 动画不明显卡顿
- [ ] `npm run build` 通过

## 9. 产品判断

这一轮是必要的，因为当前版本只是“像手机形状的网格”。

小红书爽感需要的是：

> 具体物品显现 + 存钱动作有反馈 + 完成时有海报感。

如果 V2 做完后仍然不够爽，再考虑：

1. 加真实 SVG 手机插画
2. 加更强的碎片飞入路径
3. 做分享海报
4. 做自定义目标图片

但本轮先不要贪，先把“固定手机目标”做漂亮。
