## 1. 背景

上一轮安卓真机验收中，Edge Android 出现过以下问题：

> 点击“添加到手机”后显示正在安装，但桌面看不到图标。

后来确认原因不是 App 本身问题，而是 Edge / 荣耀系统中“创建桌面快捷方式”权限未开启。

用户已经手动开启 Edge 的创建桌面快捷方式权限后，重新测试结果为：

> 可以成功创建桌面入口。  
> 从桌面打开后没有浏览器导航栏，体验更接近 App。

因此，PWA / 桌面安装相关问题可以从阻塞项中移除。

## 4. 当前唯一明显体验问题

PWA 安装问题关闭后，当前最值得修的是：

> 小金额存钱时，拼图变化不够明显。

原因：

目标金额是 ¥10,000，拼图总块数是 40，因此每块碎片约等于 ¥250。存 ¥10 / ¥50 / ¥100 时不会解锁新碎片，用户可能觉得“拼图没动”。

这不是逻辑 bug，而是反馈设计不足。

### Task 2：增强小金额存钱反馈

实现“下一块碎片进度”。

建议在 `PuzzleView` 下方增加：

```txt
下一块碎片正在路上：XX%
再存 ¥X，就能拼回下一块
```

计算逻辑：

```ts
pieceValue = targetAmount / totalPieces
currentPieceRemainder = currentAmount % pieceValue
nextPieceProgress = currentPieceRemainder / pieceValue
amountToNextPiece = pieceValue - currentPieceRemainder
```

注意：

- 达到 100% 后不显示“还差”
- 数值要格式化成 `¥80`
- 文案要轻，不要像理财软件
- 不要破坏当前 UI 风格

---

### Task 3：未解锁时增加预热反馈

当用户存钱但没有解锁新碎片时：

- 下一块未解锁碎片轻微闪一下
- 或拼图区域轻微 glow
- 或显示一个小光点飞向下一块

目的：

> 即使没有解锁完整碎片，也要让用户感觉这笔钱“喂到愿望里了”。

---

### Task 4：调整存钱成功文案

当前存钱成功文案可以进一步区分：

如果本次解锁了新碎片：

```txt
叮！新碎片回来了
```

如果本次没有解锁新碎片：

```txt
这笔钱正在把下一块碎片推近
```

要求：

- 不要删除原有文案库
- 可以新增文案组
- 根据 `newlyUnlockedPieceIndexes.length` 判断使用哪类文案
