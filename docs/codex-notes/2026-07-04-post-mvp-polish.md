## 3. 需要优先修复的问题

### 3.1 问题描述

当前 MVP 中，`app/page.tsx` 里有 `prevUnlocked` 状态，但没有真正用于 `PuzzleView`。

同时，`PuzzleView` 目前只接收：

```ts
type PuzzleViewProps = {
  totalPieces: number;
  unlockedPieces: number;
};
```

这会导致一个问题：

当用户一次存入较大金额，例如 ¥500、¥1000，可能一次性解锁多块碎片，但组件无法准确知道“本次新增的是哪几块”。

结果可能是：

- 只对最后一块做动画
- 或者已有碎片重复动画
- 或者刷新页面时误判已有碎片为新碎片

这个问题会影响产品核心体验。

核心体验必须是：

> 每次存钱后，用户能明确看到“这次因为我存钱，新拼回来了这些碎片”。

## 4. Codex 需要执行的任务

### 任务 1：修复新增碎片动画逻辑

请修改 `app/page.tsx` 和 `components/PuzzleView.tsx`。

目标：

1. 在用户存钱前，计算旧的解锁碎片数。
2. 在用户存钱后，计算新的解锁碎片数。
3. 得到本次新增解锁碎片的 indexes。
4. 将 `newlyUnlockedPieceIndexes` 传给 `PuzzleView`。
5. `PuzzleView` 根据 indexes 判断哪些碎片是本次新增。
6. 本次新增碎片依次播放动画。
7. 已经解锁但不是本次新增的碎片不要重复播放入场动画。
8. 页面初次加载 localStorage 数据时，不要把已有碎片误判为新碎片。

建议数据结构：

```ts
const newlyUnlockedPieceIndexes = Array.from(
  { length: Math.max(newUnlockedPieces - oldUnlockedPieces, 0) },
  (_, index) => oldUnlockedPieces + index,
);
```

`PuzzleView` props 建议改为：

```ts
type PuzzleViewProps = {
  totalPieces: number;
  unlockedPieces: number;
  newlyUnlockedPieceIndexes?: number[];
};
```

动画判断：

```ts
const isNewlyUnlocked = newlyUnlockedPieceIndexes.includes(i);
```

动画要求：

- 新增碎片使用 `scale`
- 新增碎片使用 `opacity`
- 新增碎片可以使用 `y`
- 新增碎片可以使用轻微 `rotate`
- 多块新增碎片应该有 stagger / delay
- 不要让所有已解锁碎片每次都重新动画

### 任务 2：增加重置本地数据按钮

为了方便测试和演示，请增加一个轻量的重置入口。

要求：

1. 按钮放在页面底部。
2. 不要太显眼，避免破坏主体验。
3. 点击后必须二次确认。
4. 确认后调用 `resetAppState()`。
5. 页面恢复默认目标、金额 0、空历史、空成就。
6. 重置后 localStorage 也要清空。
7. 按钮文案可以是：`重置测试数据`。

建议样式：

- 小字号
- 灰色
- 放在历史记录下面
- 不要使用主要按钮样式
